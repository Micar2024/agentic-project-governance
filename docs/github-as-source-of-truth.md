# GitHub as Source of Truth

Use GitHub issues, pull requests, and release notes as the public project record.

## Recommended sources

- Issue: objective and task packet
- Branch: implementation workspace
- PR: changed files, review packet, evidence, risk notes
- Release notes: accepted change summary

## Avoid

- Treating chat history as the only source of truth
- Hiding acceptance criteria in private messages
- Merging without visible evidence
- Using private context in public examples
